Solstice Edge Server (SES)

Solstice Edge Server (SES) is a virtual machine that is installed and run on an organization’s internal network to provide a single point of access for Solstice Pods to communicate with the internet. It is released as a virtual machine OVF file to be run within your VMware environment.

SES also includes Mersive’s lightweight network application Solstice Discovery Service (SDS), which allows end users, whether they be employees or guests, to efficiently discover available Solstice Pods displays using their Solstice app.

Since Solstice Discovery Service (SDS) is now bundled into SES, customers no longer need to run SDS on a separate Windows computer or server on their network. Simply configure your Pods to communicate with SES instead. Click hereClick here for more information about SDS functionality.

Why Use SES?

Solstice Pods require internet access to update to the latest software version, as well as to reach the licensing server. Internet access is also required to manage Pods through Solstice Cloud, Mersive's cloud-based management portal.

Some organizations may have security concerns about or policies that prohibit individual Pods accessing the internet directly. SES solves this problem by allowing Pods to use SES as a proxy to the internet. An organization only needs to allow a single SES instance access to the internet, and all Solstice Pod traffic will be routed through that one access point. By default, SES can only access the domains on the internet required by Solstice. (See Proxy Configuration for details.)

Solstice Edge Server Requirements

SES is delivered as a virtual machine (VM) in an Open Virtualization Format (OVF) and consists of two files that can be opened in a VMware environment. SES appears as an Ubuntu machine on your network, which can be configured to your needs.

VM Requirements for SES

The VM must be run on a machine with VMware vSphere 6.5 or later. The VMware virtual hardware should be version 13 or higher.

When the VM is run, it is currently configured to use:

  • 4 CPUs
  • 4 Gb RAM
  • 50 Gb storage

Network Requirements for SES

Mersive recommends that SES be given, at minimum, a 10 Gbps internet connection. This will be used primarily for Pods to access the internet for software updates, as well as for Solstice Cloud Management, if applicable. The amount of traffic generated by these uses is generally low except when a software update is being downloaded. Bandwidth used by Solstice content-sharing does not need to be considered since it is not routed through SES.

Open Network Ports for SES

For Solstice Pods to communicate with SES, the following TCP ports must be open on the internal network:

  • 22- Needed for SSH access.
  • 3128- Used for proxy communication from Pods to SES.
  • 53100 - Used to update information from SES to Pods.
  • 80, 53200, 53201, 53202 - Used for SDS traffic between Solstice Pods, Solstice app, and SDS.

How To Run the Solstice Edge Server VM

To start the SES virtual machine with VMware, there are two basic steps:

  1. Import the SES .ovf file provided by Mersive into VMware.
  2. Start the VM.

See the VMware documentation for details.
