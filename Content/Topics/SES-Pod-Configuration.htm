Configure Solstice Pods for SES

Solstice Pods should be configured to use the Solstice Edge Server (SES) by configuring them to use SES for SDS (Pod discovery) and proxy services. These can changes can be made in Solstice Dashboard or through a Pod’s local configuration panel (accessed by attaching a USB keyboard/mouse to a USB port on the back of the Pod).

Configure the SDS Host

Setting a Pod’s SDS Host to the SES server's IP address makes that Pod known to SDS (Solstice Discovery Service) running within SES. This makes the Pod discoverable to users in the Solstice app.

In the screenshot below, a Pod named Phobos is being configured in the Solstice Dashboard > Network tab. The IP address of the SES (in this example, 192.168.1.26) is entered in SDS Host 1 in the Display Discovery section. Checking List display to Solstice Directory Service allows this Pod’s display to be easily found by users in the Solstice app.

Configure the http and https Web Proxy

In the same Network tab of Dashboard are also the Web Server Proxy settings, where you can set proxies for http and https traffic. A Web Proxy IP Address and Web Proxy Port must be specified. If your proxy requires a login, also enter a Login Name and Password. The port used for the SES proxy is 3128. The Exclusion List and Don’t use the proxy server for local addresses on the same subnet can be left blank because these types of configuration are specified in SES itself.
