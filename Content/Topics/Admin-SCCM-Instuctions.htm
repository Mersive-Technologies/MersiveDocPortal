Deploy Solstice User Apps with SCCM

The SCCM installer package of the Mersive Solstice app allows an IT administrator to centrally and remotely deploy the app for users on a Windows computer. General deployment details can be found below, and Mersive Support can help answer any questions or concerns about specific situations.

Starting in Solstice 5.5, the installer for the Mersive Solstice app has been separated from the Solstice Conference drivers installer. Use the information below to first install the Mersive Solstice app (SolsticeClientSetup-5.5_SCCM.exe). You can then centrally deploy the Solstice Conference drivers (SolsticeConferenceDrivers-1.3.msi) with MSI.

Silent installs using SCCM require that the installer knows what actions to take on options offered by the installation wizard. You can record a data file once with the desired set of actions and use it to silently install the Mersive Solstice App across your deployment. The option to run in silent mode (/s) requires a separate data file containing preferences for those actions. Note that administrator or system privileges are needed to install system-wide, but the installer does not explicitly check for these privileges.

Silent Install Instructions

  1. Download the SCCM version of the Mersive Solstice app installer from mersive.com/download-admin > Deployment Management.
  2. Run the installer on a machine once via a command window using the command line option /r. This will run through the wizard and record a response file that will be used later. Note that the command must be run with SYSTEM or administrator rights. Example command:

    3. SolsticeClientSetup-5.5_SCCM.exe /r

  3. Run through the install wizard and set your desired configurations.

  4. Once the install wizard is completed, a file is created with the selected configurations recorded. The file is called 'setup.iss' and is located in the system’s Windows folder (e.g., C:\Windows\setup.iss).

    The setup.iss file should be removed from C:\Windows\ between recording runs since the SCCM installer appends to, rather than overwrites, the response file.

  5. If you also want to silently uninstall the Mersive Solstice app, run the installer again to uninstall, using the command line option /r to create an uninstall response file. Select the Remove option and uninstall as normal.

    Note that the SCCM installer will remove previous versions of the Mersive Solstice app before installing the latest version. An uninstall response file should only be needed to silently remove the Mersive Solstice app from devices without an upgrade or reinstall.

  6. Save a copy of the 'setup.iss' file with a meaningful name that includes the purpose and app version. Once all your response file(s) have been created, copy them to the system you want to install on, along with the Mersive Solstice app installer file (e.g., SolsticeClientSetup_SCCM-5.5.exe).
    •  Once created, response file(s) can be used multiple times for the version of the installer the were created with. However, they may need to be re-created new versions of the Mersive Solstice app installer.
    •  Uninstall response files should only be used to uninstall matching versions of the app.
  7. To silently install the Mersive Solstice app with an SCCM response file, run the following command in a command window. This example uses a response file located in the top level of the C:\ drive. Modify the path to your response files as needed.

    SolsticeClientSetup-5.5_SCCM.exe /s /f1"c:\setup_install_5-5.iss"

    The /f1 option allows the response file to be in any directory, rather than being restricted to the same directory as the install package. Mersive recommends using an absolute path to specify a response file's location, since using a relative path can produce unpredictable results.

  8. To silently uninstall the Mersive Solstice app with an SCCM response file, run the following command in a command window. As in the example above, modify the path to your response files as needed.

    SolsticeClientSetup-5.5_SCCM.exe /x /s /f1"c:\setup_uninstall_5-5.iss"
