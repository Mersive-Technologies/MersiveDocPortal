Configure Solstice Pods for SES

Solstice Pods must be configured to use the Solstice Edge Server (SES) in the following two ways. This can be done via Solstice Dashboard or through the Web Configuration tool on the Pod itself.

Configure the SDS Host

Setting a Pod’s SDS host to the SES IP address makes that Pod known to the Solstice Discovery Service within SES, making that Pod discoverable to Solstice clients.

In the screenshot below, a Pod named Phobos is being configured in Solstice Dashboard > Network. The IP address of the SES (in this example, 192.168.1.26) is entered in SDS Host 1 in the Display Discovery section. Checking List display to Solstice Directory Service allows this Pod’s display to be easily found by Solstice clients.

Configure the http and https Web Proxy

In the same Network tab of Dashboard, the Web Server Proxy settings appear. Set your proxies for http and https traffic here. A Web Proxy IP Address and Web Proxy Port must be specified. If your proxy requires a login, also enter a Login Name and Password. The port used for the SES proxy is 3128. The Exclusion List and Don’t use the proxy server for local addresses on the same subnet can be left blank because these types of configuration are specified in SES itself.
