Solstice Video Room Integration

Starting in Solstice 5.5, Solstice Pods can automatically switch between a dedicated Zoom Room, Microsoft Teams Room, or Cisco Webex Room and the Solstice Pod's multi-source wireless content sharing and bring-your-own-meeting (BYOM) video conferencing support. This configuration can also support switching between Solstice and certain all-in-one video conferencing appliances.

System Components & Requirements

The following components and related configurations are required to integrate Solstice with a computer-based video room system:

  • Gen3 Solstice Pod running Solstice 5.5 or higher with an active Solstice Subscription
    • Configure the Pod with a static IP address or hostname that can be resolved by the video room system device.
  • Device running Zoom Room or Microsoft Teams Room software
    • Download and install the Mersive lightweight app that detects camera usage and turns off the Solstice overlay.
  • Supported USB switcher that accepts RS-232 or HTTP-based API commands
    • Configure any devices to be signaled by Solstice via API commands, such as the USB switch, to have a static IP on the same network as the Solstice Pod.
    • Set the USB switch to route to the Zoom Room or Microsoft Teams Room device by default.
  • Optional USB to RS-232 cord (for RS-232 USB switch)
  • Display monitor, video, and audio peripherals

When the video room system, display monitor, and USB audio and video peripherals are connected to the Solstice Pod via a USB switching device, Solstice can be configured to signal the USB switch with RS-232 or HTTP-based API commands to toggle between and the video room system and Solstice's wireless sharing and BYOM video conferencing as needed. A second configuration is also possible to integrate Solstice with a supported all-in-one video conferencing appliance, such as the Poly Studio X30, that contains a camera and speakerphone and also accepts R-232 or HTTP-based API commands. In this type of integration, the all-in-one appliance is connected to the Solstice Pod via HDMI-in and USB. The Solstice Pod can then be configured to send API commands to toggle the all-in-one device between its native functionality and an external device mode that can pass AV input and output to the Solstice Pod.

How To Configure Solstice for Video Room Integration

Use these steps to configure a Solstice Pod for integration with a Microsoft Teams Room, Zoom Room, or Cisco Webex Room.

Step 1: Connect Solstice Pod to Room System and Peripherals
  1. Attach the Solstice Pod to power and to the local Ethernet and wireless networks, as appropriate for your Solstice deployment.
  2. The video room system (Zoom, Teams, etc.) device should also be powered on and connected to Ethernet on the same network as the Solstice Pod.
  3. Connect a display monitor to one of the Solstice Pod’s HDMI outputs (HDMI 1 or HDMI 2 on the back of the Pod). Mersive does not recommend using dual displays in a VRI configuration at this time.
  4. Connect the video room system device's HDMI output to the HDMI-in port on the front of the Solstice Pod.
  5. Connect any peripheral cameras, speakers, and/or microphones to the USB switch's USB output ports.
  6. Connect one of the USB ports on the back of the Solstice Pod to a USB input port on the USB switch.
  7. Connect a USB port on the video room system device to a USB input port on the USB switch.
  8. Optional: If using an RS-232 USB switch, connect the other USB port on the back of the Solstice Pod to the RS-232 port on the USB switch using a USB–to–RS-232 cord.
 Step 2: Configure Solstice for Dynamic Switching with Video Room System
  1. Open the Video Room Integration configuration page and enter login information if required.
  2. Check Enable Video Room Integration and select your Room System Type.
  3. Select the Device Switching Protocol based on the type of USB switch being used. If using an RS-232 switch, select RS-232.
    1. In Command to Select Solstice, enter the RS-232 command that corresponds with the port Solstice connects to on the USB switch.
    2. In Command to Select Room System, enter RS-232 command corresponding with the port the video room system connects to on the USB switch.
  4. If using a USB switch that accepts REST API commands, select HTTP/HTTPS.
    1. Under Command(s) to Select Solstice, enter an Order, Verb, Headers, Payload, and URL as needed to create an API command that will instruct the USB switch (or all-in-one device) to switch to the port connected to the Solstice Pod.
    2. Click Add HTTP Command to add rows for additional API commands as needed.
    3. Click the Delete (trash can) icon at the end of a command row to remove the associated API command.
    4. Under Command(s) to Select Room System, use the tools described in steps a–c to create an API command that will signal the USB switch to switch to the port connected to the video room system device.
  5. Click Apply at the bottom of the configuration page to save your changes.
 Step 3: Configure Solstice Connection Message on Video Room System
  1. Open the Video Room Integration configuration page and enter login information if required.
  2. If Enable Video Room Integration is not selected, first complete the directions in Step 2 above.
  3. In Instructions Text Overlay section at the bottom of the page, enter the Solstice connection instructions you would like to appear to users on the video room system display. The variables [RoomName], [ScreenKey], [WifiNetworkName], [WifiIP], [EthNetworkName], and [EthIP] will automatically be replaced with specific details for that Solstice Pod.

    For example, entering the overlay text "To access Solstice, connect to [WiFiNetworkName], choose [RoomName], and enter screen key [ScreenKey]" would generate connection directions similar to:

    To access Solstice, connect to Company_Network, choose Large Conference Room, and enter screen key 4868

  4. Select the Text Overlay Position where you would like the Solstice connection instructions to appear on the video room system screen.
  5. You may also choose a light or dark background from the Color Scheme menu.
  6. Click Apply at the bottom of the configuration page to save your changes.
 Step 4: Install Mersive ?? App on Video Room Device

The Mersive ?? app runs on the video room device to hide the Solstice connection message when a video conference is in progress. This lightweight application is quick to install and configure and should have no impact on video room performance.

  1. Download the Mersive ?? application from mersive.com/download-admin.
  2. Run [filename] on the video room system computer or NUC device to install the application. It is designed to run in the system tray on startup. Trying to start the application when it is already running will produce an error message.
  3. When the ?? application is running, a camera icon appears in the system tray, which turns green when a connected camera is in use.

  4. Click the camera icon in the system tray and select Configure.

  5. Enter the static IP address of the Solstice Pod integrated with the video room system in Pod Hostname and close the configuration box. The application will now signal Solstice to turn off its instructions overlay when a connected camera is in use.
 Step 5: Set HDMI Input Mode to 'Persistent Post'

Follow the directions below to use either Solstice Cloud or Solstice Dashboard to set the HDMI-in behavior mode to Persistent Post for the Solstice Pod integrated with a video room system. In a video room system configuration, the video room display is configured as an HDMI-in post to the Solstice Pod. In Persistent Post mode, Solstice hides the room system post when users connect wirelessly to the Solstice Pod and reinstates it when all users have disconnected from Solstice.

In Solstice Cloud

  1. In Solstice Cloud, use the left sidebar to navigate to Manage > Pods.
  2. Click the name of the Gen3 Solstice Pod to be integrated with the video room system.
  3. Scroll down and expand the Features configuration section. If a template has previously been assigned to the Pod, click Edit to change the configuration of the selected Pod.
  4. Under HDMI Input Behavior, select Persistent Post.
  5. Click Save as Unassigned to save the configuration for the selected Pod only.

In Solstice Dashboard

  1. In Solstice Dashboard, select the Gen3 Solstice Pod to be integrated with the video room system from the list of Your Solstice Instances.
  2. Go to the Appearance and Usage tab and scroll to Usage and Feature Management.
  3. Under HDMI Input Mode, select Persistent Post.
  4. Click Apply.
 Step 5: Disable Digital Signage on Solstice

Follow the directions below to use Solstice Cloud or Solstice Dashboard to disable digital signage on the Solstice Pod integrated with a video room system. In this type of integration, it is recommended that digital signage instead be enabled on the video room system.

In Solstice Cloud

  1. In Solstice Cloud, use the left sidebar to navigate to Manage > Pods.
  2. Click the name of the Gen3 Solstice Pod to be integrated with the video room system.
  3. Scroll down and expand the Digital Signage configuration section. If a template has previously been assigned to the Pod, click Edit to change the configuration of the selected Pod.
  4. If Enable Digital Signage is selected, uncheck it. The digital signage settings will become unavailable.
  5. Click Save as Unassigned to save the configuration for the selected Pod only.

In Solstice Dashboard

  1. In Solstice Dashboard, select the Gen3 Solstice Pod to be integrated with the video room system from the list of Your Solstice Instances.
  2. Go to the Digital Signage tab.
  3. If Enabled is selected, uncheck it. The digital signage settings will become unavailable.
  4. Click Apply.

Other VRI Configuration Recommendations

Mersive also recommends the following video room system configuration options to achieve the best results with your Solstice video room integration:

Zoom Room Configuration

  • To avoid receiving alerts each time Solstice-enabled conferencing is used, disable device error notifications on the Zoom Room device (Alert > Client Alert Settings).
  • Set power mode for Zoom Room device to never sleep and never suspend display, except for times when the room won’t be used for any meetings, such as at night.

Microsoft Teams Room Configuration

  • Set power mode for Teams Room device to never sleep and never suspend display, except for times when the room won’t be used for any meetings, such as at night.

Cisco Webex Room Configuration

  • Any similar Cisco recommendations?

Poly Studio X30/X50 Configuration

  • Turn off sleep mode on the Poly Studio device.

USB Switch Requirements and Tested USB Switches

USB Switch Requirements

  • USB 2.0 or 3.0 (or higher) compatibility depending on protocol version required by connected video/audio peripherals
  • Supports 2 or more host devices (Solstice Pod and video room device or all-in-one video appliance)
  • Provides commands (RS-232 or API) that set the active host device

All-in-One Video Appliance Requirements

 Supported USB–to–RS-232 Converters

Based on publicly available information, the following USB-to-serial converter chips should be supported:

  • FTDI FT232R, FT232H, FT2232H, FT4232H, FT230X, FT231X, FT234XD
  • Prolific PL2303
  • Silicon Labs CP2102 and all other CP210x
  • Qinheng CH340, CH341A

These chips can be found in many generally available USB–to–RS-232 converters. Contact Mersive Support for more information.
