Solstice Cloud Analytics: Usage

The Analytics > Usage page provides both graphical representations of and granular source data for usage history metrics collected by the Solstice Pods in your organization about Solstice-enabled collaboration sessions. See Example

The Usage page is divided into two parts:

  • Usage History Graph - The top of the page displays a graph that shows up to two types of usage data for Pods managed in your Solstice Cloud organization on a time line. You can select the type(s) of analytics data to display on the graph and what unit of time to display it in from options in the upper right and lower left of the graph. The time period selected from the options above graph also affects the Pods table below.
  • Pods Usage History Table - Below the graph, a table appears containing the usage history data details for your Pods managed in Solstice Cloud. By default, all Pods in the table are selected to display their data in the graph. The table can be filtered to display a subset of Pods using either the Search bar at the top of the table or the Filter button at the top of the page. Any filters you apply will also affect the data that is displayed in the graph.

Collected Data

Solstice data is uploaded to Solstice Cloud on a daily basis. Once the data from a Solstice Pod has been uploaded to Solstice Cloud, it is then deleted from the Pod. Solstice Cloud tracks the following usage data for Solstice Pods.

Pod Usage Data Room Occupancy Data

If your Pod is connected to a USB camera and has Room Occupancy enabled (Features settings), the room occupancy data can be collected. Graphs display total numbers for the Pods selected in the table over the time period selected at the top of the page, graphed by hours, days, weeks, or months, based on the menu in the graph's upper right corner. Pods data tables list usage data for the selected time period by individual Pod.

The following room occupancy data categories are available:

  • Average Hourly Occupancy records the average number of people detected in a Pod's meeting space each hour. Hour increments start at the top of the hour—for example, 1:00 p.m. to 2:00 p.m. If a meeting runs from 1:45 p.m. to 2:15 p.m., the number of people in the room will be averaged into both the 1:00 p.m. and 2:00 p.m. calculations.
  • Peak occupancy records the maximum number of people detected in a meeting space during a meeting.

How To

See step-by-step instructions in the sections below to learn how to access the Usage History analytics for your Solstice deployment and use available controls and filters to view the data that is most meaningful to you.

Access Usage History Analytics
  1. Log in to Solstice Cloud Management portal.
  2. In the left sidebar menu of Solstice Cloud, go to Analytics > Usage.
  3. In the top right of the Usage History page, select a time period for the usage data you want to view.
    1. You may select a pre-set time period (from 48 Hours to 90 Days).
    2. Or click Custom to choose the start and end dates of a custom time period from a calendar.
  4. A Usage History graph displays at the top of the page. The data displayed in this graph is determined by which Solstice Pods are selected in the Pods table below the graph.
  5. Use the following options to manage selections in the Pods table:
    1. Select or deselect rows for individual Pods manually using the boxes in the first column.
    2. Use the Search box at the top of the Pods table to filter Pods by Pod name.
    3. Use the Filter button at the top of the page to filter Pods by Categories defined for your Solstice Cloud Management organization.

    Filtering and changing selections in the Pods table will also change the data graphed above. The Usage History graph displays only data for Pods that currently appear in the Pods table.

  6. In addition to changing Pod table selections, you can change the graph view in a couple ways:
    1. Select Hours, Days, Weeks, or Months in the top right corner of the graph to show the data graphed by that increment of time.
    2. Select up to two usage data categories from the menus below the lower left corner of the graph to compare data for the selected Pods over the same time period. For example, you might select Content Per Meeting from the first menu, then select Total Users from the second menu. You can use this graph to compare how the total amount of posted content relates to the total amount of users over a given time period.

 Set a Custom Time Period for Usage Data Select Data Categories to Graph

Up to two Pod usage and room occupancy categories may be selected to graph on the Usage History analytics page.

  1. Locate the two categories menus below the graph. You must select one category from the left menu (blue) to display as a data line on the graph.

  2. You may select a second category from right (green) category menu to display and compare that data on the same graph with your first selected category.
  3. Or select None in the right (green) menu to show only data from the category in the left (blue) menu.
  4. Data lines for the categories you selected will display in the graph. They appear color coded to the menus they were selected from.
  5. You may also hover over the data point(s) for a time period in the graph to see the specific date(s) and numbers for each selected category displayed on the graph.
