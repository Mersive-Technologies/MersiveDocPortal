Solstice Security Features Overview

A Solstice Pod is a network-attached device that provides straightforward and secure wireless access to existing display infrastructure by leveraging a host IT network. Configuring Pods according to the guidelines laid out in this and linked topics allows users to quickly connect and share content to the displays in Pod-enabled rooms while still maintaining network security standards. Pods that are not configured properly can be vulnerable to user and network security breaches, including unauthorized user access, screen capture and recording, unauthorized changes to configuration settings, and denial-of-service attacks.

Mersive recommends that initial configurations for Solstice Pods take place on a standalone network prior to being deployed on your enterprise network. This ensures that Pods have the recommended baseline security features enabled before being exposed to critical functions and potential vulnerabilities on a larger network. To do so, the Solstice Dashboard management tool will need to be installed on a secure Windows computer and run on the same network as the Pods being configured.

Pod security options:

  • Password protect Pod configuration
  • Disable local Pod configuration
  • Serve Solstice app via port 443 (prevent man-in-the-middle redirects on insecure networks)
  • Redirect http Pod addresses to https hostname
  • Disable Pod's captive portal checking (to determine internet access)
  • Encrypt network traffic between Pods and user devices (TLS)
    • Upload (additional) custom TLS certificate for network encryption
  • Enable Screen Key on Pod (Cloud > Features template)
  • Enable Moderator Mode for collaboration sessions (default) (Cloud > Features template)
  • Disable/enable browser look-in feature (Cloud > Features template)
  • Disable network broadcast of Pod display name

Other security recommendations:

  • Initially configure Pods with Solstice Dashboard on standalone network
  • Block traffic (enable firewall) between Pods' wired and wireless networks in dual-network configuration (Dashboard only??)
