Configuring the Solstice App with CFPreferences on macOS

Preferences for the Solstice app on macOS can be configured using the Settings screen in the Solstice app or by configuring a Solstice plist (property list) using the macOS command line utility defaults.

Solstice App Settings for macOS

The following application options can be set using the defaults utility.

Both the Solstice app and defaults write to the plist. If a user changes a setting in the application, a value that was set using defaults may be overwritten, and vice-versa.

Option Type Description
PROFILE/AUTOSHAREDESKTOP Bool If set to TRUE, Solstice will automatically share your desktop to the Solstice display once you connect to the Pod (corresponds to Settings > Control > Quick Share Desktop).
PROFILE/AUTOSHAREAUDIOWITHDESKTOP Bool If set to TRUE, Solstice will automatically stream your desktop audio to the Solstice display when you share your desktop (Settings > Control > Auto share audio with desktop).
PROFILE/CALENDAR_CONFIG/REFRESH_TOKEN Bool If set to TRUE, an Office 365 calendar has been integrated with the Solstice app. You can set the value to FALSE to terminate the calendar integration. However, the calendar integration needs to be configured in the Solstice app (Settings > Conference) to authenticate with Office 365.
PROFILE/CPUWARNINGCHECKCOUNT Uint32 Reserved.
PROFILE/CPUWARNINGCHECKINTERVALMS Uint32 Reserved.
PROFILE/CPUWARNINGFAILPERCENTAGE Uint32 Reserved.
PROFILE/CPUWARNINGTOTALUSAGETHRESHOLD Uint32 Reserved.
PROFILE/HIDEUIWHENSHARING Bool If set to TRUE, the Solstice app will be automatically hidden when you share your desktop (Settings > Control > Hide desktop on share).
PROFILE/ISFIRSTRUN Bool When the Solstice app is first installed, this option is set to TRUE. This causes the Solstice app to start with a "get started" screen that prompts the user to enter a profile name. After the app is first run, this value will be set to FALSE.
PROFILE/ISFIRSTTIMEINMINUI Bool When the Solstice app is first installed, this option is set to TRUE. This causes the app to display a "got it" prompt the first time the application window is automatically hidden or minimized. (See PROFILE/HIDEUIWHENSHARING.) After the app is first run, this value is set to FALSE.
PROFILE/ISFIRSTWALKTHROUGHRUN Bool When the Solstice app is first installed, this option is set to TRUE. This causes the Solstice app to display a "walk through" animation about how to use the app to connect to a Solstice display. After the app is first run, this value is set to FALSE.
PROFILE/LOCATIONNUDGECOUNT UInt32 This option tracks the number of times the user has been prompted to enable location-based discovery of the nearest Solstice displays. When set to a value greater than 10, the prompt is disabled.
PROFILE/LOCATIONSERVICESTOGGLE Bool If set to TRUE, location services are enabled (Settings > Control > Solstice Location Services) to locate the nearest Pods. If set to FALSE, location services are disabled. (The user must also grant permissions to use location services.)
PROFILE/MANUALMRUDISPLAYS String ???
PROFILE/MANUALSDSADDRESS String If your organization has implemented the Solstice Discovery Service (SDS), set this value to the SDS IP address. The Solstice app then uses the Solstice Discovery Service Address (Settings > General) to show nearby available Solstice displays.
PROFILE/NAME String Set the value to how you would like your to appear in a collaboration session. Use quotation marks around the value (such as "First Last").
PROFILE/NOTIFICATIONLEVEL Uint32

Set which notifications you would like to receive:
0 – Do not display any notifications.
1 – Display only critical notifications.
2 – Display all notifications.

PROFILE/PINGWARNINGCHECKCOUNT Uint32 Reserved.
PROFILE/PINGWARNINGCHECKINTERVALMS Uint32 Reserved.
PROFILE/PINGWARNINGFAILPERCENTAGE Uint32 Reserved.
PROFILE/PINGWARNINGLATENCYTHRESHOLD Uint32 Reserved.
PROFILE/RELAYSHORTCUT String Enter a value to set the Conference HotKey shortcut (Settings > Conference) to Cmd+Opt+[value] for quickly opening the Start Conference box without having to go to the system tray. The default value is 'C', which corresponds to the shortcut Cmd+Opt+C.
PROFILE/RELAYSHOULDSHAREDISPLAY Bool If set to TRUE, the Share Pod Display setting for Solstice Conference (Settings > Conference) is enabled, allowing a Solstice display to be shared with remote conference participants. If set to FALSE, Share Pod Display is disabled, which may improve performance on some devices.
PROFILE/RELAYWARNINGDURATION Uint32 Reserved.
PROFILE/RELAYWARNINGINTERVALMINS Uint32 Reserved.
PROFILE/UUID String Reserved.

Examples of Using Defaults on macOS

Here is an example of how to read the values in the Solstice app plist:

$ defaults -currentHost read com.mersive.solstice.client

If a settings option is not currently set in the plist, you can create it by specifying the key type and key value. For example, to configure the Solstice app to automatically share the desktop with the display when it connects, you could enter this:

$ defaults -currentHost write com.mersive.solstice.client "PROFILE/AUTOSHAREDESKTOP/TYPE" Bool
$ defaults -currentHost write com.mersive.solstice.client "PROFILE/AUTOSHAREDESKTOP/VALUE" T

Here is an example of how to change a configure setting for the profile name to Jerry:

$ defaults -currentHost write com.mersive.solstice.client "PROFILE/NAME/VALUE" "Jerry"

Here is an example of how to delete a configured setting in the Solstice app plist:

$ defaults -currentHost delete com.mersive.solstice.client "PROFILE/AUTOSHAREDESKTOP/TYPE"
$ defaults -currentHost delete com.mersive.solstice.client "PROFILE/AUTOSHAREDESKTOP/VALUE"

The Solstice app reads the plist settings on start up. If you change plist values using the defaultscommand while the app is running, the app will not show the change on the Settings page. However, the app will behave in accordance with the new plist setting.

For example, if the plist setting for AUTOSHAREDESKTOP is FALSE, then the Solstice app will show that Quick Share Desktop is OFF. However, if the value is changed to TRUE using the defaults command, the Settings page does not change; however, when the app connects to the display the desktop will be automatically shared. To force the Solstice app to change what is displayed on the Settings page, you will need to exit and restart the app.
