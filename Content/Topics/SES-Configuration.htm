Configure the Solstice Edge Server (SES)

Once the SES virtual machine is up and running on your network, you can log in and configure it. (See the SES Introduction for SES requirements and how to run SES in VMware.)

Logging in to Solstice Edge Server

SES can be logged in to in a couple of ways. You may use the terminal window of the VMware environment, or if you know the IP address (or hostname) of SES, you can connect to it remotely via SSH.

Common SSH clients are PuTTY, MobaXterm, or Git Bash. In Windows 10, the OpenSSH client can be installed as an optional feature.

Type the following into your SSH client to connect to SES:

ssh admin@<IP address or hostname of SES>

When logging in to SES, the username will always be ‘admin’. Initially, the password for the admin user is ‘Solstice’:

solstice-edge-server login: admin
Password: Solstice

You will be prompted to choose a new password immediately upon logging in. As always, choose a strong password and keep it safe.

Running sesadm to Configure SES

The sesadm tool is used to configure SES. Because sesadm requires elevated privileges, you will need to use sudo to run it with super user permissions:

sudo sesadm

When using sudo, the server will occasionally prompt you to reenter the admin password you used to log in to SES.

You will see the menu for making all SES configuration changes:

No configuration changes will be applied until you enter option 4 to exit sesadm. You will be asked if you want to apply the changes upon exiting.

1 - Network Configuration

Enter 1 to select Network Configuration. This option allows you to define how SES networking is set up.

SES can be configured to use DHCP, where SES’s IP address will be automatically assigned by network equipment, or Static IP, where you can assign SES a fixed IP address. If using Static IP, you will be prompted for the IP address to use, the network prefix length, the Gateway, and the DNS address.

 2- Proxy Configuration

The Proxy Configuration section contains several options:

1 - Parent Proxy Configuration

For added security, some organizations may want Solstice Edge Server, which itself acts as a proxy, to operate inside another proxy server or servers. This option can be enabled and disabled in Parent Proxy Configuration. Enabling a parent proxy forces all SES traffic to the outside internet, including traffic from Solstice Pods, through the specified proxy.

There are several parent proxy configuration options. Some simple examples are provided in the configuration file, including single-parent, multi-parent, and round-robin parent proxy configurations. SES’s proxy is built on the Squid proxy platform; thus all Squid options are available to you. For detailed Squid documentation, click here.

When you enter the option to edit parent proxy settings, the GNU nano editor appears. This is a simple text editor with editing options shown at the bottom of the screen. Help is available in the editor, or from GNU nano.

When your configuration is complete, use Ctrl-X to exit the editor and save your changes.

Remember, settings changes are not applied until you exit sesadm and confirm doing so.

2 - MAC Access List

If the MAC Access List is enabled, SES can be accessed only by Pods whose MAC addresses are specified in the configuration file. If this feature is disabled, any MAC address, regardless of what is in the configuration file, is allowed access. (See other ways to limit network access in the Local Network Access List and Proxy Access List sections.)

Edit the configuration file to enter the MAC addresses of all Solstice Pods you want to be able to communicate to the internet through SES.

You can locate the MAC addresses of Pods in Solstice Cloud Management and Solstice Dashboard.

3 - Local Network Access List

The Local Network Access List limits by IP address which devices and networks are allowed to communicate through the SES proxy. Edit the list to specify the IP addresses you want to have access to SES (IPV4 and IPV6 are supported). The network that the Solstice Pods are using should be listed here. You may both specify whole networks using CIDR notation and list IP addresses individually. (See other ways to limit network access in the MAC Address List and Proxy Access List sections.)

In Squid proxy configurations (the proxy platform used by SES), use the commenting character ‘#’ before text you want the proxy to ignore. Information after the ‘#’ (to the next line break) can remain in the configuration file without taking effect. For more about Squid configuration, click here.

4 - Proxy Access List

The Proxy Access List allows you to configure which internet sites SES will be allowed to access.

By default, SES allows Pods access to only the sites shown below. These are generally the only sites Pods need to access, for the latest software releases and to verify licenses. Removing these is not recommended. (See other ways to limit network access in MAC Address List Configuration and Local Network Access List.)

  • .mersive.com - Solstice Pods check for software updates and manage Pods via Solstice Cloud Management.
  • manager.flexnetoperations.com - License server for Solstice Pods subscriptions.

  • www.google.com - Pods verify internet access through the proxy configured on SES.

There are only a couple cases in which organizations may need to open up additional domains. If you use the Digital Signage or RSS feed features of Solstice, add the domains of the URLs your Pods use to access these features to the Proxy Access List configuration.

The content shared to Solstice Pod displays by both employees and guests via the Solstice client does not leave the Pod. It never goes to SES, nor will it ever be passed through the SES proxy.

 3 - Export sesadm Settings

Enter this option in sesadm to back up all of your current SES configurations to a TAR archive file. You can then copy this file to your organization’s backup drives. When there are new SES releases, Mersive will deliver a new virtual machine OVF file. Simply “power down” the previous SES virtual machine, “boot” the new SES virtual machine, and import the settings. The new SES release is now operational, and Pods will start sending discovery information to SDS.

Import functionality is not part of the SES v0.5 beta release. Either import or an option for automatically preserving settings through upgrades will be included in future SES releases. Manual reconfiguration of SES due to an upgrade will not be necessary.

Exit sesadm

From the main sesadmmenu, enter option 4 to exit the administration tool and apply your configuration changes. If you choose to not apply changes, none of the settings you made during your current sesadm session will be saved.

Entering y will immediately apply your network and proxy configuration settings. There is no need to do anything else for the settings to take effect.

Basic Current Settings with sesinfo

After saving configuration changes in sesadm, the system displays some basic SES settings: SES version, IP address, and proxy port number. (See screenshot in the Exit sesadm section above.)

To see this information anytime, enter:

sesinfo
