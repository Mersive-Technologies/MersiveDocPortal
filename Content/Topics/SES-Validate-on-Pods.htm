Validate SES on Solstice Pods

There are a variety of ways to test that your Solstice Edge Server (SES) is set up correctly. Typing sesinfo in the SES command prompt displays the SES version, IP address, and proxy port number. Solstice Dashboard can also be used to check if things are working as expected.

Is SDS on SES Working?

In addition to configuring Pods, Solstice Dashboard can be used to view any Pods currently reporting their presence to the Solstice Discovery Service (SDS). This allows you to check whether Pods are configured correctly and confirm the that SDS instance running within SES is making Pods discoverable to the Solstice user app.

Is Proxy on SES Working?

Solstice Dashboard can also be used to check that the SES proxy settings are correctly allowing Pods to reach the internet. Selecting a Pod and clicking the Test Proxy Settings button (in the Web Server Proxy section) will prompt the Pod to attempt both http and https connections to www.google.com.

Green Success results show that the Pod has successfully communicated with Google through the SES proxy, and the parent proxy, if configured.

You can also test the negative case, that Pods are not able to access sites they should not have access, to in a couple of ways:

  1. Use sesadm to remove or comment out the www.google.com line from the Proxy Access List configuration file. (See Configure SES for details on using sesadm.) Exit sesadm to save configuration changes and try Test Proxy Settings in Solstice Dashboard again. This should produce Failed results.
  2. Similarly, remove or comment out the .mersive.com entry (used for updates) from the Proxy Access List in sesadm. Then go to the Licensing tab of Dashboard, select a Pod, and click Check for Updates. Dashboard should report it is unable to check for updates rather than providing the status of the Pod’s software.

Ensure that SES is running before performing the tests above. The SES server being powered down will also cause Pods to be unable to communicate through the SES proxy.
