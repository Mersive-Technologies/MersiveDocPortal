Pod Location Configuration Settings

Location features in Solstice Cloud use GPS estimates that are learned from Solstice Location Services to automatically group your Pods into categories for Country, City, and Building. These categories are just like other categories, except that they are populated automatically and don’t require editing. This allows a user to walk into a room, open the Solstice app, and see the Pod they are physically closest to at the top of the display list. If the automatic categories aren't intuitive enough for users, you can assign a “friendly name” for each building which end users will see when they open the Solstice app.

Additionally, for administrative purposes, Solstice Cloud admins can easily segment groups of Pods based on their location while managing template configurations, viewing usage analytics, monitoring uptime, and much more. Admins can use an interactive map to see how Pods are clustered together based on their location, including fine tuning the location of a Pod down to a few feet or even identifying which floor of the building the Pod is on.

The use of Solstice Location Services requires the following:

  • Pods must be imported into Solstice Cloud
  • Pods need location services enabled (Manage > Templates > Room Intelligence)
  • Pods must be running Solstice version 5.1 or later

Pod Location Views

There are two different ways to view the locations of Solstice Pods included in your organization in Solstice Cloud. These views can be found in Solstice Cloud under Manage > Location.

In the map that appears, the following views are available in the bottom left corner:

  • Cluster by Pods shows all Solstice Pods in your organization clustered by location. Click on a Pod cluster to zoom in on Pods in that location.
  • Cluster by Buildings shows Solstice Pods in your organization clustered by building assignment. Zoom in to view finer building location details, including individual Pod locations within buildings (within two stops of maximum zoom).

  • Pods not assigned to a location will be represented on location maps using an estimated initial location.

Clicking the dot representing an individual Pod will display that Pod's configuration settings. You can then perform the following location-related actions from the Pod configuration page:

  • Set the initial location of a Pod
  • Change the location of a Pod
  • Fine-tune the location of a Pod
  • Add a new building location
  • Name a Pod for its location

How To

Set the Initial Location of a Pod

If a Solstice Pod has never been assigned to a location, its configuration page will prompt you to assign the Pod to a building.

  1. In the left sidebar of Solstice Cloud, navigate to either Manage > Pods or Manage > Location.
  2. In the Pods table that appears, open the Pod configuration page by clicking the name of the Pod you want to assign a location to.

    If you manage a large number of Pods, you can reduce the number of Pods displayed in the table by entering all or part of the Pod name in the search box or by setting a filter.

  3. The map at the top of the Pod Configuration Settings page will indicate the Pod's location is not set, and a Set location box will appear.
    1. From the Select building menu, choose the appropriate building for the selected Pod and click Next.

    2. If the desired building is not in the list, click Add New Building and follow the prompts. (See details on how to add a building below.) You can then select it from the list.
    3. Click Next.
  4. The location is now set for the Pod, and the map will zoom in on the Pod's location. From here, you can view and change or fine-tune the location of a Pod.
 Change the Location of a Pod

If a Solstice Pod has previously been assigned to a location, the bottom of the map on its configuration page will display the building, floor, city, and country for that Pod. The building and floor selections can be modified.

  1. In the left sidebar of Solstice Cloud, navigate to Manage > Location.
  2. Scroll down to the Pods table, and click the name of a Pod that has an assigned location in the Building column.

    Click the Building column header twice to sort Pods already assigned to buildings to the top of the table.

  3. Below the map of the Pod's location, location details appear. Click into the Building field without clicking the dropdown arrow. The field will change to a search box, and you can start typing the name of the desired building in the field.

  4. From the filtered building list that appears, select the desired building.
  5. If the desired building is not in the list, click Add New Building and follow the prompts. (See details on how to add a building below.) You can then select it from the list.
  6. The map will zoom in on the Pod's new location.
  7. Select the appropriate Floor for the Pod, as needed. If the desired floor does not appear in the menu, click Add New Floor and enter the floor number. Then click Save.
 Fine-Tune a Pod's Location

After a Solstice Pod has been assigned to a location, the Pod's configuration page will show a zoomed-in map of that location. If the Pod appears in the wrong place in the building, its location can be fine tuned.

  1. In the left sidebar of Solstice Cloud, navigate to Manage > Location. Scroll down to the Pods table, and click the name of a Pod that has an assigned location in the Building column.

    Click the Building column header twice to sort Pods already assigned to buildings to the top of the table.

  2. On the map of the Pod's location, click Fine-tune Location in the upper right. The cursor will change to a hand icon.
  3. Click and drag in the map area to reposition the map until the Pod's location dot appears in the correct location in relation to the building.
  4. Click Save.

After you have fine-tuned a Pod location, Location Services will no longer attempt to refine the location of the Pod.

 Add a Building Location

If the building where a Solstice Pod is located does not appear in the existing list of buildings in Solstice Cloud, you can easily add it.

  1. Follow the directions above for setting or changing a Pod's location assignment. If the desired building doesn't appear in the Building menu, click Add New Building.
  2. In Building Address, start typing the address of the building to be added. From the list of possible matching addresses that appears, select the correct address.

  3. If desired, enter a reference name for the new building (e.g., Headquarters) in Building Name.
  4. Click Save.
 Name a Pod for Its Location

Mersive recommends assigning Pods names that correspond with the location or room the Pod is in, such as North Conference Room, to help meeting participants easily identify the appropriate Solstice Pod display for their location.
